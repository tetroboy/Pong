# Pong
Pong game on sfml

Реалізація класичної гри Pong на C++ з використанням бібліотеки SFML 3.0. Гра має головне меню з вибором рівня складності, гру проти бота з ШІ, паузу, екран перемоги/поразки, звуки удару та голу. Проект структурований за папками для зручності (States, Entities, Systems, Utils). Збірка здійснюється через CMake, протестовано на Windows (Visual Studio 2022) - гра компілюється та запускається без додаткових кроків (run.bat або запуск exe).

1. Знання мови C++ (не C)
Весь проект написаний на сучасному C++17: розумні вказівники (std::unique_ptr), лямбди, constexpr, override, range-based for. 

2. Знання ООП та вміння його використовувати

Базовий клас Entity з віртуальними методами (src/Entities/Entity.h).
Спадкування: Paddle -> Entity, PlayerPaddle та BotPaddle -> Paddle.
Поліморфізм: вектор std::vector<std::unique_ptr<Entity>> у PlayingState - методи викликаються поліморфно.
Інкапсуляція: приватні члени (position, velocity) з геттерами/сеттерами.

3. Вміння використовувати STL

Контейнери: std::vector<std::unique_ptr<Entity>> у PlayingState.
Розумні вказівники: std::unique_ptr для всіх ігрових об’єктів та текстів (через відсутність default constructor у SFML 3.0).
Алгоритми: std::clamp для обмеження позиції.
std::to_string для рахунку.

4. Акуратно оформлений структурований код
Проект поділений на папки:

States/ - стани гри
Entities/ - ігрові об’єкти
Systems/ - менеджери (фабрика, рахунок, UI)
Utils/ - константи

6. Вміння підключати та використовувати зовнішні бібліотеки
Використано SFML 3.0:

Підключено через CMake (find_package(SFML)).
Графіка, ввод, аудіо: sf::RenderWindow, sf::Keyboard, sf::Sound, sf::Music.
Ресурси завантажуються безпосередньо в класах (шрифт, звуки).

7. Знання архітектури ігрових рушіїв

Game::run() - poll events -> update -> render -> display (src/Game.cpp).
Обробка подій відокремлена (handleEvents у кожному стані).
Update та render розділені.
Патерн State для перемикання між меню, грою, паузою та кінцем гри.

8. Використання патернів проектування

State - основний патерн для станів гри (src/States/, changeState у Game.cpp).
Factory Method - EntityFactory для створення сутностей (src/Systems/EntityFactory.cpp).
Observer - ScoreManager сповіщає UIManager про зміну рахунку (src/Systems/ScoreManager.cpp).
Strategy - різна поведінка бота залежно від складності (через конструктор BotPaddle).

Singleton не використовувався — замість нього пряме завантаження ресурсів у потрібних класах.

9. Код компілюється та запускається без додаткових кроків

Протестовано на Windows 10/11 з Visual Studio 2022 та SFML 3.0 (Win64).
Збірка через CMake.
Ресурси копіюються автоматично після збірки.
Запуск: run.bat або exe з папки Release.
Для Ubuntu: не протестовано через відсутність світла на ПК autorun для лінукса повинен працювати корректно але встановлення сфмл 3.0 поки перекладено на користувача

10. README з посиланнями
Структура проекту:

src/main.cpp - точка входу
src/Game.cpp - ігровий цикл та стани
src/States/ - стани гри
src/Entities/ - ігрові об’єкти
src/Systems/ - фабрика, рахунок, UI
src/Utils/Constants.h - глобальні константи
resources/ - шрифт, звуки

Запуск: зібрати через CMake, запустити exe. Керування: W/S - гравець, ESC - пауза. Гра до 10 очок.
